# V7 Architecture

## Modules

- `VideoIngestionService`: store upload or authorized YouTube source.
- `VideoNormalizationService`: convert source into stable working MP4.
- `WorkingMediaResolver`: always picks normalized working media first.
- `VideoMetadataService`: FFprobe metadata.
- `AudioExtractionService`: extract WAV from working video.
- `RealAiProvider`: transcribe using OpenAI/local Whisper.
- `TranscriptIntelligenceService`: semantic/rule-based analysis from transcript.
- `ClipGenerationService`: persist real clip candidates and subtitle segments.
- `FfmpegVideoRenderer`: render MP4 1080x1920.
- `TrendManager`: modular trend provider.

## Data Sources

Final output may only use:

- uploaded/or authorized source file,
- FFprobe metadata,
- FFmpeg video/audio signal,
- OpenAI/local Whisper transcript,
- OpenAI semantic JSON or rule engine over real transcript,
- user edits.

No dummy result should be shown as final production output.

## Queue Flow

`AnalyzeVideoJob`:

```text
uploaded_video.normalized
→ transcribe
→ classify
→ generate copy pack
→ generate clip + subtitle
→ project.clips_ready
```

`RenderClipJob`:

```text
clip + subtitle + preset
→ FFmpeg render
→ rendered_videos.ready
→ project.render_status completed
```

## Storage Layout

```text
storage/app/private/projects/{project_id}/uploads/original.ext
storage/app/private/projects/{project_id}/normalized/source-{video_id}.mp4
storage/app/private/projects/{project_id}/thumbnails/source-{video_id}.jpg
storage/app/private/projects/{project_id}/audio/source-{video_id}.wav
storage/app/private/projects/{project_id}/renders/{clip_id}-{platform}.mp4
```

Exact paths are generated by service classes and can be moved to S3-compatible storage via Laravel filesystem.
